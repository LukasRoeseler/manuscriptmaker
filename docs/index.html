R REPLICATION RESEARCH · Publishing Workflow

From “accepted” to “published”, automatically.

A single-source publishing pipeline that turns an accepted manuscript — in Word, Markdown, Quarto, R Markdown, or LaTeX/Overleaf — into HTML, a typeset PDF, archival JATS XML, and an Open Journal Systems import package, in the journal's exact design. Hosted entirely on GitHub — no server to maintain.

⚡ Instant preview (in browser) 📤 Submit a manuscript 🎨 Customize for another journal 📦 Repository

One source, four outputs

Write or upload once. Quarto + Pandoc render HTML, PDF (LaTeX), and JATS from a single canonical file; a script assembles the OJS package.

Any input format

.docx .doc .md .rmd .qmd .tex and Overleaf .zip. Word citations from Zotero/Mendeley are converted to BibTeX.

Pull-request workflow

Submit by opening a PR. A bot posts a live preview and galleys. Proofreading happens as PR comments; merging publishes the article.

Reusable theme

The journal design is a Quarto extension parameterized by one theme.yml. Other journals reuse the engine unchanged.

The pipeline

submit → normalize.py → article.qmd (+ references.bib + figures/) │ quarto render ────┼────────────┬───────────────┐ r2-html r2-pdf r2-jats → enrich └──────── build_ojs.py ──────────┘ OJS native-import XML
